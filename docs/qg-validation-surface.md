# Q-G Validation Surface

## Purpose

This note summarizes the public-safe validation surface for Q-G final architecture and AI Brain Map integration.

## Required gates

Required gates protect the main architecture shape.

- App shell boundary check
- Route module binding check
- Brain module contract check
- Documentation consistency check
- Public release boundary check

## Optional gates

Optional gates can be added after the required surface is stable.

- Visual regression check
- Deployment readiness summary
- Version ledger summary
- Example structure validation
- Bundle surface review

## Manual evidence

Manual evidence can remain private or be summarized publicly.

- Screenshots
- Browser replay logs
- Full validation reports
- Historical package evidence
- Internal handoff notes

## Public rule

The public repository should describe gate purposes and boundaries. It should not publish full private logs, full build artifacts, or deployment configuration.
